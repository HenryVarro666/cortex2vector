## Data Creation

### create_3hinge_adj_feature_matrix

- [ ] 1

这段代码被用来处理和分析从一个 Excel 文件中抽取的数据，特别是处理关于人脑头皮图像的信息，除此之外还有一些关于这些数据的属性或特征。

首先，它定义了一些函数，它们根据不同目标执行不同的任务：

- get_individual_connection_info： 这个函数读取一个 Excel 文件，并创建了两个字典来储存装饰节点的信息，一个储存节点特征，另一个储存节点间的连接。

- create_adj：该函数基于装饰节点之间的连接创建一个相邻矩阵，然后根据该连接创建一个相邻矩阵，然后计算并保存1跳，2跳和3跳相邻矩阵。

- read_labels：从文本文件读取并返回所有的标签列表。

- create_label_index_dict：创建一个字典，将标签映射为索引，以及保存这个映射到一个文本文件中。

- create_one_hot_feature：这个函数使用装饰节点的特征和标签索引字典创建一个独热编码的特征，并保存到一个文本文件中。

在代码的主要部分，首先创建了所需的一些目录，然后读取所有主题和球的列表，并对每个主题和球执行以下操作： - 从 Excel 文件中获取单个连接信息。 
- 创建相邻矩阵。 
- 创建一种独热编码特性。

这段代码主要用于处理与头皮图像相关的信息，并生成特征矩阵。代码主要分为以下几个部分：

2. create_adj 函数：该函数基于获取的连接信息创建相邻矩阵，并计算并保存1跳、2跳和3跳的相邻矩阵。

3. read_labels 函数：该函数从文件中读取标签，并返回标签列表。

5. create_one_hot_feature 函数：该函数使用标签索引字典和节点特征字典创建独热编码的特征，并将其保存到文件中。

在主函数部分，首先设置输入和输出目录，并创建必要的目录。然后，通过调用上述函数处理每个主题和球体的数据：

首先，通过读取相关文件，获取节点特征和连接信息。
然后，使用连接信息创建相邻矩阵，并计算1跳、2跳和3跳的相邻矩阵。
接下来，使用标签索引字典和节点特征字典创建独热编码的特征，并保存到文件中。
这段代码主要用于处理和转换头皮图像相关的信息，并提取出有用的特征信息。

### create_3hinge_node_input

- [ ] 1

这个代码用来通过处理节点特性来生成更复杂的特性。特性可以包括原始特性（one-hot encoded），也可以包括通过相邻矩阵计算出的特性（节点的朋友/邻居节点的特性）。每个节点最终会拥有一组新的特性，它们来自于原始特性和其邻居的特性，这些都会保存在一个独立的文件中。

在主函数中，首先设置输入和输出路径，并创建必要的目录。然后，依次处理每个主题和球体配对。对于每一对，它会加载节点的一阹编码特性和相邻矩阵，然后通过将相邻矩阵与原始特性相乘来计算1-hop、2-hop和3-hop特性。最后，它会将结果保存为文件，并通过generate_node_feature函数为每个节点生成并保存一个特性矩阵。

generate_node_feature函数获取四组特性（one-hot，1-hop，2-hop和3-hop），然后将其组合在一个矩阵中。然后将这个矩阵保存到一个文件中，文件名中包含节点号和主题球体配对。

这段代码基本上是对给定的数据进行更深一层的特性工程，通过从一个节点的邻居节点获取信息，能够获取更全面的数据特性，从而在数据分析中可能获得更好的结果。

这段代码的目的是生成节点特征。它基于使用多跳邻居节点信息计算出的特征（包括原始特征和邻居节点特征）来生成每个节点的特征矩阵。

首先，它定义了一个函数generate_node_feature，该函数接受四组特征（特征数相同），然后将它们组合成一个节点特征矩阵。该函数使用一个循环遍历所有节点，对于每个节点，它将原始特征和邻居节点特征按行组合到一个4行的矩阵中。然后，它将该矩阵保存到一个文件中，文件名包含前缀和节点号。

在主函数中，首先设置输入和输出路径，并创建必要的目录。然后，依次处理每个主题和球体配对。对于每个配对，它会加载原始特征和多跳邻居特征所在的文件。然后，根据相邻矩阵和原始特征计算1-hop、2-hop和3-hop特征（节点的邻居节点特征）。最后，它将生成的特征保存为文件，并使用generate_node_feature函数为每个节点生成并保存一个特征矩阵。

这段代码的目的是对给定数据进行节点特征的生成。通过结合原始特征和节点的邻居节点特征，可以得到更丰富的节点特征矩阵，从而更好地描述数据和节点之间的关系。
